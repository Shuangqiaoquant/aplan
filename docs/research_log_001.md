# 研究日志 001：为什么 APlan 不做成荐股器

发布时间：2026-07-07  
作者：ShuangqiaoQuant

## 开场

很多量化项目最容易被问到的问题是：这个系统能不能直接告诉我明天买什么？

APlan 的回答目前很明确：不能，也不应该。

这听起来像是在降低项目吸引力，但恰恰相反。对一个仍处于研究阶段的系统来说，最重要的不是把每一个候选股票包装成机会，而是知道什么时候应该停下来，什么时候证据不够，什么时候模型看起来有效但其实经不起验证。

APlan 从第一天起就不想成为一个“自动荐股器”。它更像一个公开建设中的 A 股投研实验室：让数据、策略、公告证据、风控、模拟盘和审计记录彼此制约，而不是让任何一个模型或 Agent 单独决定答案。

## 荐股器的问题

“荐股器”这个形态有天然诱惑。

它容易传播，容易制造紧张感，也容易让读者快速产生期待。每天给出几个股票，配上一些理由，看起来就像一个产品已经完成了。

但真正的问题也在这里。

如果一个系统只输出“买什么”，却不展示数据截止时间、候选生成逻辑、失败样本、交易成本、涨跌停约束、样本外验证和风险拦截，那么它的吸引力很可能来自叙事，而不是研究质量。

A 股市场尤其不适合这样做。公告、监管、减持、业绩预告、重组、流动性和市场情绪都会改变短中期走势。单纯用技术指标追强势，很容易在回测里看起来顺滑，在真实执行里遇到买不到、卖不掉、滑点变大、行业退潮或公告风险。

所以 APlan 的第一条原则是：宁可慢，也不要把没有通过验证的东西包装成确定性。

## APlan 真正在做什么

APlan 不是要替人拍脑袋选股，而是要把投研过程拆成可以检查的模块。

目前系统关注几件事：

- 数据是否完整，是否有重复、异常和日期错位；
- 候选股票是否真的有相对强度和流动性改善；
- 公告、基本面、估值和市场环境是否提供了反证；
- 策略是否经过训练集、隔离验证集和不同调仓起点测试；
- 回测是否考虑交易成本、滑点、涨跌停、停牌和 T+1；
- 风控是否限制仓位、集中度、现金、换手和回撤；
- 每次运行是否能用报告和哈希链复盘。

这套设计的核心不是“让系统更聪明”，而是让系统更难自欺。

一个候选股票可以被记录下来，可以被观察，可以进入后续验证，但在策略没有通过隔离验证前，它不能被说成交易建议。一个回测结果可以有局部亮点，但如果调仓起点一变就失效，它就不能进入模拟盘。一个公告可以看起来利好，但如果没有历史事件验证，它暂时不能直接给策略加分。

## 为什么失败记录很重要

公开项目最容易展示的是成功曲线，最难展示的是失败。

但对 APlan 来说，失败记录是资产。

如果一个策略在训练期表现不错，到了验证期失败，这不是丢人的结果。它说明系统没有把训练集里的偶然性直接推向模拟盘。如果某一天数据质量检查失败，日报显示“今日不操作”，这也不是流程事故，而是风险闸门在工作。

真正危险的是系统永远有答案。

如果每天都能给出买入理由，每次回测都能找到漂亮参数，每个公告都能解释成催化，那么这套系统就失去了研究价值。APlan 需要保留说“不知道”“不通过”“先观察”的能力。

## Agent 应该站在哪里

APlan 会使用 Agent，但不会把 Agent 放在最终决策的位置。

Agent 更适合做三类事情：

- 把公告、报告和文本材料整理成结构化证据；
- 帮助解释候选股票背后的正反因素；
- 生成可以被人和代码复核的研究记录。

Agent 不应该直接修改历史数据，不应该改写回测结果，也不应该绕过风控生成可执行指令。

这是一条很重要的边界。Agent 的价值不在于“像人一样有直觉”，而在于它能把大量文本和上下文变成更容易验证的证据。最终，一个证据是否有用，仍然要回到历史验证、样本外表现、模拟盘记录和风险控制。

## 为什么选择公开

公开 APlan，不是为了发布每日荐股，也不是为了证明某个策略已经成熟。

公开的目标有三个。

第一，获得反馈。数据工程、回测假设、风控规则、公告分类、报告呈现，每一层都可能有盲点。公开能让更多读者指出问题。

第二，建立纪律。一个公开项目更难偷偷跳过失败记录，也更难把尚未验证的想法包装成结论。

第三，沉淀长期信用。量化研究不应该只展示收益率，也应该展示方法论、约束条件和承认错误的能力。

如果未来 APlan 有机会产品化，它也应该先成为一个可信的研究工具，再考虑订阅、私有部署、咨询服务或机构化路径。

## 当前状态

APlan 现在仍处于 `research_only` 阶段。

这意味着：

- 系统可以生成候选和日报；
- 系统可以记录观察和风险提示；
- 系统可以做回测、验证和审计；
- 系统不会默认连接真实券商；
- 策略未通过验证和审批前，不会生成可执行交易指令。

当前最重要的工作不是寻找一个看起来最漂亮的参数，而是补齐证据覆盖，改进验证流程，建立可重复的模拟盘准入标准。

## 希望读者如何反馈

如果你对 APlan 感兴趣，最有价值的反馈不是“明天买什么”，而是下面这些问题：

- 这个回测有没有未来数据泄漏？
- 这个候选生成逻辑是否过拟合？
- 这个公告分类有没有把风险误判成利好？
- 这个风控规则是否太宽或太窄？
- 这个报告是否足够清楚地表达了不可执行状态？
- 哪些数据源、字段或事件研究最值得优先补齐？

这些反馈会真正帮助项目变好。

## 结语

APlan 的长期目标不是制造更多噪音，而是建立一个能被检查、能被质疑、能从失败中更新的 A 股投研系统。

如果它最后能产生有价值的策略，那应该是纪律和验证的副产品，而不是营销话术的结果。

所以第一篇研究日志先从一句朴素的话开始：

APlan 不是荐股器。它是一套让投资研究更难自欺的工具。

> 免责声明：本文和 APlan 项目仅用于研究和教育，不构成投资建议、证券推荐或收益承诺。任何候选、回测、报告和模拟盘结果都需要独立验证。
