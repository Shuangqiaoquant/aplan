# Feedback System

Use this directory to turn public comments into project signal.

## Weekly Process

1. Collect comments from WeChat, Zhihu, Xiaohongshu, GitHub, and private chats.
2. Copy high-signal feedback into files based on `template.md`.
3. Classify each item.
4. Convert repeated feedback into:
   - documentation updates;
   - research tasks;
   - data quality tasks;
   - product ideas;
   - compliance warnings.
5. Summarize the week in `weekly_feedback_YYYYMMDD.md`.

## High-Signal Feedback

- Points out a likely data leak.
- Questions a backtest assumption.
- Identifies missing risk controls.
- Suggests a useful data source.
- Explains confusion in public-facing language.
- Flags compliance risk.

## Low-Signal Feedback

- Asks for specific stock recommendations.
- Requests guaranteed return.
- Pushes for private trading signals.
- Repeats platform-style hype language.
